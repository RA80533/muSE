Detailed Description

A frame is the local environment of a process.

Field Documentation

muse_stack _muse_process_frame_t::bindings_stack

The bindings stack is used to manage bindings during continuation invocation.

The stack structure is similar to muse_stack_t in that every entry is a cell, but there is a meaning to the entries on the stack - they are all in pairs - the even indices are all symbols and the odd indices are all their bound values.

Referenced by create_process(), fn_define(), free_process(), mark_process(), and muse_popdef().

muse_stack _muse_process_frame_t::locals

The "locals" are a set of process-local storage values.

A symbol's value is different for each process.

Referenced by create_process(), free_process(), mark_process(), and muse_intern_symbol().

recent_t _muse_process_frame_t::recent 

Each process has a "recent" list - a vector of 8

most recent calculations performed.

The recent list is accessed using the "the" primitive. The 0, 2, 4, etc. indiced contain what recent item to access and the 1, 3, 5, etc. indices contain the recent item.

Referenced by create_process(), free_process(), mark_process(), muse_add_recent_item(), muse_find_recent_item(), muse_find_recent_lazy_item(), muse_pop_recent_scope(), and muse_withdraw_recent_item().
